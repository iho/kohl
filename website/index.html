Three pillars

What the chain hides — and how

Every transfer is self-authenticating. There is no account balance, no nonce, and no signed origin for user payments. The CLSAG is the authorization.

01

Sender anonymity

Each input is a ring of 16 outputs — 15 decoys plus the real spend. CLSAG proves knowledge of one private key and a matching key image without revealing which ring member was spent.

Hides which past output moved

02

Receiver privacy

Dual-key CryptoNote stealth addresses. Senders derive one-time destinations; only the view key can detect payment, only the spend key can unlock it. View tags keep wallet scanning fast.

Hides who was paid

03

Amount confidentiality

Amounts live inside Pedersen commitments. A public fee anchors the balance equation; aggregated Bulletproofs prove every output amount is in range — no overflow minting.

Hides how much moved

Design choices

Cash only. Fair launch. Stable rules.

  • No contracts, no EVM, no assets palette. The runtime is deliberately small: system, timestamp, RingCT, difficulty.
  • Zero genesis supply. Every coin is mined. RandomX PoW avoids a premine or sale for the initial stake.
  • Ristretto everywhere. Prime-order group avoids historical cofactor / small-subgroup footguns.
  • Native host crypto. CLSAG and Bulletproof verification run natively so block times stay practical under WASM.
  • Fees to miners. Public fee with a min fee-per-byte floor; carried into the next coinbase.

Get started

Run a private cash node

shell 
cargo build -p kohl-node --release

# Throwaway miner keys (printed at startup):
./target/release/kohl --dev --validator --tmp

# Persistent payout (same seed as kohl-wallet):
./target/release/kohl --dev --validator --tmp \
  --mining-seed <64-hex>
