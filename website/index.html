Three pillars

What the chain hides — and how

Every transfer is self-authenticating. There is no account balance, no nonce, and no signed origin for user payments. The membership proof is the authorization.

01

Sender anonymity

Each spend proves the real input is one of the full set of mature outputs under a published Merkle root — not a wallet-chosen ring of decoys. Wallets do not sample mixins.

Hides which mature output moved · interim set size ≤ 64

02

Receiver privacy

Dual-key CryptoNote stealth addresses. Senders derive one-time destinations; only the view key can detect payment, only the spend key can unlock it. View tags keep wallet scanning fast.

Hides who was paid

03

Amount confidentiality

Amounts live inside Pedersen commitments. A public fee anchors the balance equation; aggregated Bulletproofs prove every output amount is in range — no overflow minting.

Hides how much moved

Design choices

Cash only. Fair launch. Honest limits.

  • No contracts, no EVM, no asset palette. The runtime is deliberately small: system, timestamp, RingCT, difficulty.
  • Zero genesis supply. Every coin is mined. RandomX PoW — no premine for the initial stake.
  • FCMP-only spends. One production path from genesis. No dual era of fixed-size rings on mainnet history.
  • Ristretto everywhere. Prime-order group avoids historical cofactor / small-subgroup footguns.
  • Native host crypto. Membership and Bulletproof verification run natively so block times stay practical under WASM.
  • Interim scale limit. Today’s proofs are O(n) over the mature set, capped at 64. Larger trees need a later log-size construction — we do not claim unlimited anonymity sets yet.

Tokenomics

Monero-like emission, perpetual tail

Smooth decaying curve toward a long-run supply target, then a permanent tail reward so the security budget never hits zero.

Network privacy

Dandelion++ on by default. Tor still recommended.

On-chain privacy does not hide your IP. Kohl stems transactions before flooding them; operators should still route the node (and wallet RPC) over Tor for defence in depth.

01

Dandelion++

Local transfers enter a stem path on /kohl/dandelion/1, then fluff into ordinary gossip (~10% per hop, embargo fallback).

Built into the node

02

Outbound Tor

There is no --proxy-server flag. Use torsocks and bind P2P to localhost so dial-out goes through SOCKS.

Level 1 in the runbook

03

Onion services

Tor hidden service → 127.0.0.1:30333, advertise /onion3/… with --public-addr. Optional RPC onion for wallets.

Level 2–3 in the runbook

shell · outbound via Tor 
torsocks ./target/release/kohl \
  --chain kohl-ash --validator \
  --listen-addr /ip4/127.0.0.1/tcp/30333 \
  --rpc-port 9944 \
  --network-backend libp2p \
  --mining-seed <64-hex>

Get started

Run a private cash node

shell · public seed (mainnet) 
cargo build -p kohl-node --release

# One-time: node key + data dir
./scripts/setup-seed.sh

# Public seed — open TCP 30333; set your IP
PUBLIC_ADDR=/ip4/YOUR.PUBLIC.IP/tcp/30333 \
MINING_SEED=<64-hex> \
  ./scripts/run-seed.sh

# PeerId for bootnode string:
./target/release/kohl key inspect-node-key --file ./data/seed/node-key
# → /ip4/YOUR.PUBLIC.IP/tcp/30333/p2p/<PeerId>

# Optional: ship a chainspec with bootnodes embedded
./scripts/make-chainspec.sh --chain kohl \
  --bootnode /ip4/YOUR.PUBLIC.IP/tcp/30333/p2p/<PeerId> \
  --output chainspecs/kohl.json

Miners connect with

shell
BOOTNODE=/ip4/…/tcp/30333/p2p/… \
MINING_SEED=<64-hex> \
  ./scripts/run-miner.sh

# or: --chain ./chainspecs/kohl.json
